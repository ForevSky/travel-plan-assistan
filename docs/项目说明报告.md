## 1. 项目概述

旅行规划小助手是一款基于 **DeepSeek 大语言模型** 的**旅行规划垂直领域**应用，提供 **CLI 命令行** 与 **Web 平台** 两种使用方式。

用户通过自然语言描述需求（如「我想去杭州两日游，轻松一点」），系统生成包含行程、美食、交通、预约收费清单、注意事项等在内的结构化攻略，并支持追问式微调、流式输出、后台继续生成、切换对话后续订、暂停展示、多会话管理、分享与导出。

> **重要说明**：本系统输出为 AI 规划建议，预约、收费、天气等信息均非实时 API 查询结果，出行前请务必核实官方公告。

## 3. 功能说明

### 3.1 核心功能（CLI / Web 共用）

#### （1）自然语言旅行规划

- 输入示例：「成都玩 3 天，想吃火锅看熊猫」
- 系统自动解析目的地与天数，生成 9 模块完整攻略

#### （2）追问式微调

- 在已有攻略基础上继续对话，例如：
  - 「再加一个人」
  - 「不想去刘公岛」
  - 「推荐便宜一点的酒店」
- 原则：**最小范围改动**，仅修订相关章节

#### （3）领域拒答

- 对编程、理财、闲聊等非旅行问题拒答
- 旅行相关的天气、季节、穿衣等问题正常处理
- 双层机制：规则词库 + LLM 意图解析

#### （4）攻略导出

| 端 | 方式 | 输出 |
|----|------|------|
| CLI | 输入 `Y` 或「保存」 | `storage/output/城市_x日游_YYYYMMDD.txt` |
| Web | 攻略卡片导出按钮 | 浏览器下载 TXT |
| Web | 追问输入「保存」 | 写入 `storage/output/` |

### 3.2 Web 平台专属功能

#### （1）多会话管理

- 侧栏展示会话列表，按时间分组（今天 / 昨天 / 近 7 天 / 更早）
- 支持搜索过滤
- 支持删除会话
- **生成中可切换对话**：前端断开 SSE，后端在独立线程继续生成并落库
- 切回原对话时自动检测 `generating` 状态并续订流式输出

#### （2）流式对话、后台生成与续订

- SSE 实时展示 AI 生成过程
- 生成任务由 `generation_registry` 在后台线程执行，**客户端断开不中断**
- 切换对话后再切回，自动调用 `GET /messages/stream` 续订，从当前进度继续展示
- 生成中底部按钮可暂停**前端展示**；后台仍会完成并写入数据库
- 切回或刷新后可看到完整回复

#### （3）分享

- **会话分享**：分享完整问答记录
- **攻略分享**：分享单条完整攻略
- 生成分享链接并复制到剪贴板
- 独立分享页 `/share/:token` 只读预览

**分享展示规则**：

- 生成进行中不展示分享入口
- 最后一条 AI 回复被暂停时不展示分享入口
- 重新发送并成功完成后恢复分享能力

#### （4）结构化攻略展示

- 攻略以文档卡片形式呈现
- 每日路线时间轴、美食列表、关联图示（上图下词）
- 非攻略类消息以对话气泡展示

### 3.3 输出内容清单

| 序号 | 模块 |
|------|------|
| 1 | 行程概览 |
| 2 | 季节与天气适配建议 |
| 3 | 每日合理路线与攻略 |
| 4 | 美食推荐清单 |
| 5 | 景点预约与收费汇总清单 |
| 6 | 交通出行建议 |
| 7 | 注意事项与必备物品 |
| 8 | 预算参考 |
| 9 | 免责声明 |

## 4. 技术架构

### 4.1 架构图

```
                    ┌─────────────┐
                    │  Web 浏览器  │
                    └──────┬──────┘
                           │ HTTP / SSE
                    ┌──────▼──────┐
                    │  FastAPI    │  backend/web/api/
                    │  + SQLite   │  storage/data/
                    └──────┬──────┘
                           │
              ┌────────────┼────────────┐
              ▼            ▼            ▼
        chat_service  generation_registry  share_service  plan_service
              │              │                  │
              └──────────────┴──────────────────┘
                           ▼
                    domain / llm_client
                           │
                           ▼
                      DeepSeek API

        CLI: main.py → application → services → domain → llm_client
```

### 4.2 技术选型

| 类别 | 选型 | 说明 |
|------|------|------|
| 后端语言 | Python 3.8+ | CLI 与 Web 共用业务逻辑 |
| Web 框架 | FastAPI + uvicorn | REST + SSE 流式 |
| 数据库 | SQLite | 会话、消息、分享 token |
| 大模型 | DeepSeek Chat | OpenAI 兼容 API |
| 前端框架 | React 18 + TypeScript | Vite 构建 |
| UI 组件 | Ant Design 5 | 企业级组件库 |
| 路由 | React Router 6 | SPA 路由 |
| Markdown | react-markdown | 攻略内容渲染 |

### 4.3 关键设计决策

1. **自然语言优先**：统一对话式交互，取消多菜单分功能模式  
2. **解析与生成分离**：先结构化 city/days，再生成攻略  
3. **领域双层守门**：规则层零成本拦截 + LLM 层语义判断  
4. **修订最小化**：追问只改相关模块，保留会话内完整攻略  
5. **Web 复用 CLI 核心**：`chat_service` 调用 `plan_service`，避免双份业务逻辑  
6. **后台生成解耦**：`generation_registry` 将 LLM 生成与 SSE 连接分离，支持切换后续订  
7. **前端 AbortSignal**：切换/暂停时断开 SSE，不影响后台任务完成

## 5. 部署与使用

### 5.1 CLI 模式

```bash
cd travel-plan-assistant
pip install -r requirements.txt
copy .env.example .env   # 填入 DEEPSEEK_API_KEY
python main.py
```

### 5.2 Web 开发模式

```bash
# 终端 1：后端
python web_server.py

# 终端 2：前端
cd frontend
pnpm install
pnpm dev
```

访问 http://localhost:5173

### 5.3 Web 生产模式

```bash
scripts\build-frontend.ps1
python web_server.py
```

访问 http://127.0.0.1:8000

### 5.4 典型 Web 使用流程

```
1. 打开首页，输入：我想去杭州两日游，轻松一点
2. 自动创建会话并进入聊天页，流式生成攻略
3. （可选）生成中切换到其他对话 → 后台继续生成
4. 切回原对话 → 自动续订，继续看到输出（或已完成的完整攻略）
5. （可选）点击暂停按钮停止前端展示（后台仍会完成）
6. 继续追问：不想去灵隐寺 → 系统微调攻略
7. 点击攻略卡片「分享」或顶部「分享会话」复制链接
8. 点击「导出」下载 TXT；或输入「保存」写入服务端 output 目录
9. 侧栏可切换、搜索、删除历史会话
```

### 5.5 配置项说明

| 文件 / 变量 | 用途 |
|-------------|------|
| `.env` | `DEEPSEEK_API_KEY` 等（勿提交 Git） |
| `backend/core/config.py` | 业务常量、领域词库、Token 限制 |
| `UVICORN_RELOAD` | `0` 关闭后端热重载（生产） |
| `storage/data/conversations.db` | Web 会话数据（运行时生成） |
| `storage/output/` | 导出攻略 TXT |

## 6. 产品边界与合规

### 6.1 能力边界

| 支持 | 不支持 |
|------|--------|
| 单城市 1–7 天短途规划 | 跨国多城市联程 |
| 季节/出行天气适配建议 | 实时天气 API 查询 |
| 预约/收费常识推断 + 汇总清单 | 实时票务/库存查询 |
| 交通方式与动线建议 | 可购票班次与实时票价 |
| 预算区间估算 | 精确到分的报价 |
| 本地单实例多会话 | 用户登录、云端多设备同步 |

### 6.2 免责声明

本系统生成内容为 AI 推断结果，不构成官方预约、票价、开放时间依据。用户应在出行前查阅景区、交通、酒店等官方渠道信息。

## 8. 风险与建议

| 风险 | 影响 | 建议 |
|------|------|------|
| 模型幻觉 | 错误景点/价格 | 保持汇总清单结构；用户核实官方信息 |
| API 不可用 | 无法生成 | 重试 + 本地兜底；生产环境监控 |
| Token 成本 | 费用波动 | max_tokens 上限；缓存策略 |
| 密钥泄露 | 安全风险 | `.gitignore`；定期轮换密钥 |
| 切换对话导致生成中断 | 内容丢失 | 后台继续生成 + 切回续订（已实现） |
| 暂停后会话不完整 | 误分享不完整内容 | 前端隐藏分享入口（已实现） |
| 后台任务内存占用 | 多会话并发生成 | 单进程内存注册表；生产可考虑任务队列 |
| SQLite 单文件 | 并发与备份 | 生产可考虑迁移 PostgreSQL |
