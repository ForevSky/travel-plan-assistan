# 旅行规划小助手 — 技术方案（已实现版）

> 文档版本：v1.2 · 对应代码仓库 `travel-plan-assistant`  
> 智能能力由 **DeepSeek API** 驱动；支持 **CLI 命令行** 与 **Web 平台** 双模式。

---

## 一、项目背景与定位

### 1.1 业务痛点

| 痛点 | 表现 |
|------|------|
| 信息零散 | 景点、美食、交通、预算分散在多个渠道 |
| 决策繁琐 | 天数有限，手工排期易漏项、折返、节奏不合理 |
| 经验不足 | 不清楚预约规则、季节准备、当地消费水平 |

### 1.2 产品定位

**旅行规划垂直领域助手**，聚焦自然语言规划与结构化攻略交付：

- 用户用**自然语言**描述需求（如「杭州两日游，轻松一点」）
- 系统生成**完整攻略**（9 大模块），支持**追问式最小范围修订**
- 非旅行问题**拒答并引导**
- 支持 **CLI 本地导出** 与 **Web 多会话、流式对话、后台生成、流式续订、分享**

### 1.3 设计原则

- **领域垂直**：旅行规划专用，双层领域校验（规则 + LLM）
- **可运营**：词库、拒答文案、天数范围集中在配置层
- **容错优先**：API 重试、中文友好报错、可选本地兜底模板
- **双端复用**：Web 聊天编排复用 CLI 领域层与业务层逻辑

### 能力边界（CLI / Web 共用）

- 单城市 **1–7 天**短途规划
- 不接入实时机票/酒店/天气付费 API
- 不做用户登录与多租户（本地/单实例 SQLite）
- 输出为 AI 推断，出行前需核实官方信息

## 四、核心业务流程

### 4.1 首规划（`plan_service.run_natural`）

1. **规则层** `domain_guard.validate_new_request`：离题词拒答
2. **解析层** `intent_parser.parse`：LLM 输出 JSON → `city` + `days`
3. **生成层** `plan_service.run`：单次 API 生成 9 模块完整攻略

### 4.2 追问修订（`plan_service.run_revise`）

1. **路由** `domain_guard.is_new_trip_request`：区分「微调」与「重新规划」
2. **修订** 基于当前攻略 + 用户反馈，**最小范围**改动
3. **输出** 展示调整说明；内部更新完整攻略供后续追问与保存

### 4.3 Web 流式对话（`chat_service` + `generation_registry`）

1. 持久化用户消息至 SQLite
2. 复用 CLI 领域守门、意图解析、规划/修订逻辑
3. 通过 **SSE** 推送事件：`user_message` · `status` · `delta` · `done` · `error`
4. 前端 `fetch` + `ReadableStream` 消费（`AbortSignal` 支持客户端断开）

**后台生成与续订**

| 机制 | 模块 | 说明 |
|------|------|------|
| 后台任务 | `generation_registry` | 在独立线程中执行 `process_message_stream`，客户端断开不停止 |
| 事件缓存 | `GenerationJob.events` | 缓存全部 SSE 事件，支持多个订阅者 |
| 流式续订 | `GET .../messages/stream` | 切回原对话时从当前进度续订，重放已产生事件并继续接收 |
| 状态查询 | `GET .../conversations/{id}` | 响应含 `generating`、`generation_status` 字段 |

**典型流程**

```
用户发送消息
  → POST /messages/stream 启动后台任务并订阅 SSE
  → 用户切换到其他对话（前端 abort，后台继续）
  → 用户切回原对话
  → GET /conversations/{id} 发现 generating=true
  → GET /messages/stream 续订，恢复流式 UI
  → 后台完成 → done 事件 → 助手消息落库
```

**SSE 事件说明**

| 事件 | 载荷 | 用途 |
|------|------|------|
| `user_message` | Message | 服务端确认的用户消息 |
| `status` | `{ text }` | 规划进度提示 |
| `delta` | `{ content }` | 增量文本 |
| `done` | StreamDonePayload | 完成，含 has_plan/city/days/title |
| `error` | `{ message }` | 错误信息 |

### 4.4 暂停生成与切换对话（前端）

**暂停按钮**

- 生成中底部发送按钮切换为**暂停图标**，可点击
- 调用 `AbortController.abort()` 中断前端 fetch 流
- 已输出内容保留在 UI 中，消息取消 `streaming` 标记
- 后端任务**仍会继续**直至完成并落库；刷新或切回后可看到完整回复

**切换对话**

- 切换会话时前端同样 abort 当前 SSE 连接
- 后台生成**不受影响**，完成后写入 SQLite
- 切回原对话时，`ChatPage` 检测 `generating=true` 并自动调用续订接口
- 若切回时已完成，直接展示完整助手消息

**分享限制**

- 生成进行中不展示分享入口
- 最后一条 AI 回复被用户主动暂停时不展示分享入口
- 重新发送并成功完成后恢复分享能力

### 4.5 保存与导出

| 方式 | 触发 | 输出 |
|------|------|------|
| CLI | 追问态 `Y` / 「保存」 | `storage/output/城市_x日游_YYYYMMDD.txt` |
| Web 服务端 | 追问态输入「保存」 | 同上 |
| Web 浏览器 | 攻略卡片「导出」按钮 | 本地下载 TXT |

### 4.6 分享

| 类型 | API | 预览 |
|------|-----|------|
| 会话分享 | `POST /api/conversations/{id}/share` | `/share/{token}` 展示全部问答 |
| 攻略分享 | `POST /api/conversations/{id}/messages/{msg_id}/share` | `/share/{token}` 展示单条攻略 |

分享 token 存于 SQLite `shares` 表；链接自动复制到剪贴板。

## 五、内容输出规范（产品定稿）

### 5.1 「不做 vs 要做」边界

| 维度 | 不做 | 要做 |
|------|------|------|
| **天气** | 不调用天气 API；不写实时气温/降水 | 季节与天气适配建议（穿衣、带伞等） |
| **景点票务** | 不查实时票价/余票 | 常识推断收费/预约，**单独汇总清单** |
| **交通购票** | 不查班次/可购票信息 | 大致交通方式与动线建议 |
| **路线** | 不做 GIS 计算 | 地理顺路、节奏合理、避免折返 |

### 5.2 完整攻略 9 大模块（顺序固定）

1. 行程概览  
2. 季节与天气适配建议  
3. 每日合理路线与攻略  
4. 美食推荐清单（Top 8）  
5. **景点预约与收费汇总清单**（独立章节，必含）  
6. 交通出行建议  
7. 注意事项与必备物品  
8. 预算参考  
9. 免责声明  

### 5.3 Web 结构化展示

前端 `travel-plan` 模块解析 Markdown 攻略：

- 按章节切分（概览、每日路线、美食等）
- 每日路线渲染为**时间轴**组件
- `@img[name|category|desc]` 标签解析为**关联图示**（上图下词）
- 完整攻略消息以文档卡片样式展示，附带分享/导出操作

## 七、Web API 接口

### 7.1 会话

```
GET    /api/conversations
POST   /api/conversations          { "title": "新对话" }
GET    /api/conversations/{id}
       → 含 generating: bool, generation_status: string
PATCH  /api/conversations/{id}     { "title": "..." }
DELETE /api/conversations/{id}
```

### 7.2 聊天

```
POST /api/conversations/{id}/messages/stream
     Content-Type: application/json
     Accept: text/event-stream
     Body: { "content": "..." }
     → 启动后台生成任务并推送 SSE

GET  /api/conversations/{id}/messages/stream
     Accept: text/event-stream
     → 续订进行中的生成（404 表示无活跃任务）

POST /api/conversations/{id}/export
```

### 7.3 分享

```
POST /api/conversations/{id}/share
POST /api/conversations/{id}/messages/{msg_id}/share
GET  /api/share/{token}
```

### 7.4 健康检查

```
GET /api/health
```

## 九、异常与兜底

| 场景 | 处理 |
|------|------|
| 未配置 API Key | 启动时 `ensure_api_key()` 终止并提示 |
| 401 / 429 / 网络错误 | `llm_client` 重试 + 中文错误提示 |
| 生成失败 | 可选 `fallback/local_data.py`（成都/北京） |
| 解析失败 / 领域拒答 | 引导用户重输 |
| 修订未返回完整攻略 | 回退上一版攻略 |
| 客户端暂停 / 切换对话 | 前端 `AbortError`，断开 SSE；后台继续生成并落库 |
| 续订时任务已结束 | GET stream 返回 404，前端重新加载会话详情 |
| 分享不存在 | 分享页展示「内容不存在或已失效」 |

## 十一、风险与后续扩展

### 11.1 风险

| 风险 | 缓解 |
|------|------|
| 模型幻觉 | Prompt 约束 + 免责声明 + 汇总清单结构 |
| API 限流/网络 | 重试 + 本地 fallback |
| 密钥泄露 | `.gitignore` + 文档警示 |

| 暂停后会话不完整 | 前端隐藏分享；后台完成后刷新可见完整内容 |

### 11.2 后续扩展方向

- 用户认证与云端同步
- RAG 接入本地权威攻略文档
